Estimation

Definition

Estimation is a temporal process - things need to be integrated over time.

It aims to:

  • Come up with an estimate $\hat{x}$ for some unknown state $x$, both of which are vectors.
  • Quantify uncertainty by making a very tight distribution, so that a robot is more certain of its state. For example, it is not 100% certain to know where an object is.

Math

Consider the measurement equation: $$ Hx = y - v $$ Only the shape of the distribution w.r.t. the unknown $x$ is known. If the error $v$ is known, $x$ can be found. Or if $x$ is known, $v$ can be solved. Since neither is known, it becomes an estimation problem.

Notes

  • In general, more the measurements, the more accurate an estimate will be.
  • An estimation problem boils down to figuring out the measurement model $H$ which is often the tricky bit.

Sources of uncertainty

The real world (bot and environment) is uncertain in the following ways:

Stochastic

The effect of actions can have uncertain outcomes due to intrinsic or extrinsic factors.

A vehicle intending to follow a certain path might get blown to one side by a gust of wind or suddenly stop working. It is not possible to know when an uncertain event could occur that could then cause a state change.

Partially observable

The real worl is only partially observable. There is uncertainty about the current state of the world.

Where is the vehicle? What's behind a particular object? It is hard to know everything about the world.
