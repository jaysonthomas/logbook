GLUT stuff

Ref. It stands for openGL Utility Toolkit.

glutMainLoop

It calls the display callback over and over, calling idle in between so it can maintain a specific framerate. Ref.
  • Display callback
    • Main logic to draw whatever it is that should be going on.

  • Idle callback
    • Some very lightweight operations to determine what the change in state should be from the last time display was called to the next time. For example, if we're animating something, this would be where we change its position or orientation.
After a call to glutMainLoop(), we have no control over the execution of our program besides the callback functions we initialized before the call.
  • glutDisplayFunc - the draw function
    • Call glClear, followed by all the drawing code and then glutSwapBuffers().

  • glutIdleFunc - the update function
    • Right at the end, call glutPostRedisplay.

  • glutMainLoop
    • Loops within itself, processing events and triggering callback functions when necessary
