Apt: Setting up a new repository

Setting up the keychain

[Ref]

A GPG signing key is like a digital ID card or a passport for a software. Developers use this key to "sign" their software packages, proving that they are authentic and haven't been altered by anyone else.

APT, the package manager, has a list of trusted IDs (keys). If a software package isn't signed by a key in this list, APT will refuse to install it because it can't verify its authenticity.

Steps

  • Download and add the signing key to APT's trusted keychain.
  • Add the software repository to APT's sources list.
  • Update APT's package index.
  • Install the software package.
  • Set relevant environment variables, pointing to where the software package was installed.

Installation of the software

To find out all the dependencies 

apt-cache depends drake-dev
To filter out the non-essential but recommended packages 

apt-cache show drake-dev | grep Recommends
