Extern

Declaration


extern int a;
When declared in a header file, this tells the compiler, "There is an integer named a defined somewhere else (in another file). Don't allocate memory for it now; just trust that the linker will find it later."

This allows a global variable to be shared among multiple source (.c) files.

Delcaring in a header file allows for a centralised announcement. Any .c file that includes that header gets the announcement and can use the particular variable.

Definition


int a;
This tells the compiler to allocate memory for an integer named a. This should happen in exactly one .c file.

Linker errors

  • Multiple Definitions
    • If you write int a; (without extern) in the header file, and include that header in two different .c files, the linker will fail. It will complain that a is defined twice (multiple definition of a).

  • Undefined Reference
    • If you declare extern int a; in the header but forget to define int a; in exactly one .c file, the linker will fail because it can't find the memory address for a.
