C: Struct

Definition


    struct A {
      int b;
    };
This allows to create variables of type struct A. 

struct A obj;
obj.b = 10;

Just declaration


struct A;
This tells the compiler that A is a struct but does not define its members. Instances of A cannot be created yet as it needs to be defined somewhere else.

This is used when:

  • The definition of the struct needs to be kept private and hence defined in a file of a private scope (defined by CMakeLists.txt).
  • A is used in pointers struct A *ptr; before it is fully defined.

Alias


typedef struct A a;
It creates an alias a for struct A, so the former can be used instead of the latter.

Self referencing


typedef struct A
{
  int b;
  struct A *next;
}a;

// Another way
typedef struct
{
  int b;
}a;
The first example gives the structure a tag name struct A, allowing it to refer to itself inside its own definition (essential for linked lists), since the compiler is aware of the tag. Alias a is created right after.

The second example creates an anonymous (no tag) structure, which cannot refer to itself until the definition is complete with an alias. This is useful for simple data containers that do not need to point to other instances of themselves.
