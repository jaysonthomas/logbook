Moveit

Install moveit

  • Source ROS2 installation
  • Install rosdep which is written in python for automatic dependency installation
  • Initialise and update rosdep
  • System update

  • Install colcon extension, mixin - help with specifying multiple command line args.
  • Install python vcstool - help with cloning supporting repos for a given repo.

  • Create workspace with source folder.
  • Clone repo containing moveit tutorials into src.

  • Check that there isn't a version of moveit already installed.
  • Using rosdep, install all dependenices required by any ros package under src.

  • Go up a folder and build using the debug or release keyword. Can take an hour to compile.
  • Source the workspace (setup.bash). Add to bashrc if required.

RViz interaction with moveit

  • ros2 launch moveit tutorials demo launch.py
  • Can save it with an rviz extension.

Visualise with Rviz

  • Add dependency and header file to moveit_visual_tools.
  • Build

  • Include thread header and create an executor thread that run the main node.
    • Add this after the logger and before the main code section.

  • Add spinner.join after shutting down ros.

  • Create and initialise moveit_visual_tools with the current node, its base link and the robot model.
    • Delete all previous markers and load remote control.

  • Create closures to draw text, prompt a user and draw trajectory.
Use trigger() to refresh the screen after any drawing or prompting.

Object interaction

Inform moveit of an object, so it plans around the box.
  • Include planning_scene_interface.
  • Create collision object w.r.t. the robot's base.
  • Give the box an id, set its dimensions and pose.
  • Push_back the primitive and its pose into a list forming the collision object.
  • Add the whole collision object to the scene.
