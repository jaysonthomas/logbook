Configuration Launch

Steps

Configuration

URDF is required with a ros2_control tag to specify hardware interfaces.

A package with a config (bringup/config/.yaml) for the different controllers being used.

Launch

Create a robot_state_publisher node with the URDF (robot + controller description) as a parameter.

Create a ros2_control_node found in package controller_manager with the YAML file as parameter. This node subscribes to the /robot_description topic from robot_state_publisher node for the URDF. The node is called controller_manager.

Use spawner found in package controller_manager to create the various controller nodes. Each controller node will automatically be given the YAML file parameters that was setup in step 2.

Create an rviz node. Confirm frames.

Confirm list of nodes and topics.

Teleop using the keyboard, sending messages to particular topics.
