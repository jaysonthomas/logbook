Control manager node

Main purpose

Its primary job is to manage the lifecycle of controllers (loading, unloading, starting, stopping) using the controller_manager::ControllerManager class. 

    control_manager_node = Node(
      package="controller_manager",
      executable="ros2_control_node",
      parameters=[control_manager_yaml_path],
      output="both",
    )

Resource manager

Resource Manager is an internal C++ component of Controller Manager. Its tasks include:
  • Parses the URDF file and loads hardware interfaces.
  • Allocates memory for controller instances.
It is either passed the URDF file containing definition of the hardware interfaces, directly as a robot_description via parameters or it subscribes to the topic /robot_description which robot_state_publisher publishes to.

Controller spawner node

A controller node is a plugin that lives inside the Controller Manager. It is a temporary spawner script, not a long-running node that handles data. 

    forward_position_controller_node = Node(
      package="controller_manager",
      executable="spawner",
      arguments=[
        "forward_position_controller",
        "--param-file",
        control_manager_yaml_path,
      ],
    )
Once this node successfully sends a service request to the manager to load and start a controller, the spawner node shuts itself down. The actual math for say a position control then runs inside the control_manager_node.
