Overview

Contents

The Controller Manager is the primary ROS node that handles the lifecycle of controllers and coordinates with the hardware.

Inside the Controller Manager, there is a C++ class called the ResourceManager that is responsible for loading, initializing, and managing all hardware components defined in a URDF file.

Hardware Components are loaded as plugins (using pluginlib). They are not independent processes or nodes; they are shared libraries linked into the Controller Manager at runtime.

Details

When ros2_control is launched, a URDF's content is passed to the controller_manager node as a string parameter called robot_description.

Inside the controller_manager (specifically in its on_configure or initialization phase), it passes the string within robot_description parameter to a class called ComponentParser (found in the hardware_interface package). The parser hunts for ros2_control XML tags. It then iterates through every hardware block and extracts the plugin name and the component type.

The parser passes the plugin names to the ResourceManager, which uses pluginlib to find the compiled .so libraries corresponding to the names and get back corresponding class instances.
