Parameters flow from YAML to C++

Contents

  • Spawner node
  • The YAML file is passed to a spawner at launch time via a launch.py file.
  • The spawner node sends client requests to the controller_manager server

  • The controller_manager reads the YAML and uses the type field to look up and load the correct plugin via pluginlib.

  • The plugin instance (RobotController class) is spawned under a namespace matching the YAML key (r6bot_controller), which makes all ros__parameters accessible via the ROS parameter server under that node/namespace.

YAML file


    r6bot_controller:
      ros__parameters:
        type: ros2_control_demo_example_7/RobotController
  

    type: <package name or namespace>/<Plugin class name>
