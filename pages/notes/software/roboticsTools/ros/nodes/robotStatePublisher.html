robot_state_publisher

Overview


    robot_state_publisher_node = Node(
      package="robot_state_publisher",
      executable="robot_state_publisher",
      parameters=[
        {"robot_description": get_robot_description(package_name, urdf_rel_path)}
      ],
    )
robot_description holds the instructions to generate the entire XML content of a robot model. The instructions are obtained by running the xacro command on the urdf.xacro file path.

Debug

In a new terminal, query the running ros2 system: 

    ros2 param get /robot_state_publisher robot_description

Use

The robot_state_publisher calculates and broadcasts the 3D poses of all the links of a robot. It takes two inputs:
  • The URDF: This tells the node the "static" relationship (e.g., "The hand is 0.5 meters from the elbow").
  • Joint States: It listens to a topic (usually /joint_states) that provides the "dynamic" angles (e.g., "The elbow is currently bent at 45 degrees").
By combining these two, it calculates the position of every single part of the robot relative to the base and broadcasts them as TF transforms. It doesn't know about gravity, friction, mass, or torque. It is purely kinematic.
