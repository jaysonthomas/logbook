ROS: Installation

Verify other ROS distributions are not sourced

Make sure no other version of ROS2 or ROS1 are automatically sourced when a terminal is started. Sourced means that a setup file of a ROS2 distribution is automatically called when you open a terminal. 

printenv ROS_DISTRO
Inspect .bashrc which is a setup file that is automatically executed when a terminal is opened. 

sudo apt-get install gedit
cd ~
gedit .bashrc
The following line shouldn't be seen: the source command is used for a setup.bash file of a ROS distribution; in this case the Iron ROS2 distribution. 

source /opt/ros/iron/setup.bash

Configure ubuntu repositories

Install ubuntu universe repository. 

sudo add-apt-repository universe
sudo apt install software-properties-common
To verify: 

grep ^deb /etc/apt/sources.list

Set locale

That supports UTF-8. 

sudo apt update && sudo apt install locales
sudo locale-gen en_GB en_GB.UTF-8
sudo update-locale LC_ALL=en_GB.UTF-8 LANG=en_GB.UTF-8
export LANG=en_GB.UTF-8

Set up ROS repository sources

Add the ROS 2 GPG key. 

sudo apt update && sudo apt install curl -y

sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
Add the repository to the source list. 

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
  • deb: Tells the system this is a binary software repository.
  • arch=$(dpkg --print-architecture): Automatically detects if your system is amd64 (typical PC) or arm64 (like a Raspberry Pi).
  • signed-by=...: This is a security feature. It tells Ubuntu to only trust packages from this source if they are signed with the specific ROS digital key.

  • http://packages.ros.org/...: The actual web address where the ROS 2 files live.
  • $(. /etc/os-release && echo $UBUNTU_CODENAME) is a shortcut that checks the version of Ubuntu (like "Jammy" or "Noble") and inserts that name to get the software built specifically for the OS version.

  • The pipe (|) takes that long string and hands it to the next command:
  • tee takes the input and writes it into a new file named ros2.list.
  • > /dev/null: This just keeps things tidy. It prevents the command from printing the entire string back out to your terminal window.

Install ROS2 packages

Update the caches and the system 

sudo apt update
sudo apt upgrade
Install ROS2 

sudo apt install ros-kilted-desktop
sudo apt install ros-dev-tools

Environment variables setup

Inform what ROS2 environment variables/paths to use in a bash session (terminal): 

echo "source /opt/ros/kilted/setup.bash" >> ~/.bashrc

# For the changes to take effect
source ~/.bashrc    

# Check the line is added
gedit ~/.bashrc

Confirm ROS2 version


env | grep ROS

ROS package dependencies


sudo rosdep init
rosdep update
