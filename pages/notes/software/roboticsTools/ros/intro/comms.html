Philosophy

Overview

At the lowest level, ROS offers a message passing interface that provides inter-process communication. The ROS middleware provides:
  • Remote Procedure Calls
    • Request/response
      • Asynchronous/synchronous request/response interactions between processes. A Service is defined by a pair of messages: one for the request and one for the response.

    • Preemptable
      • If a low-priority RPC is running on the server and a high-priority RPC arrives, the system can interrupt (pause or kill) the low-priority one to free up resources immediately.

  • Publish/subscribe message passing
    • Topics are named buses over which nodes exchange messages.

  • Recording and playback of messages
