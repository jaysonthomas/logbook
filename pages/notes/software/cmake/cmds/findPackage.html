find_package

Overview

find_package is just a locator. Its only job is to scour a system, find where Eigen is installed, and grab its files, headers, and libraries. CMake deliberately separates finding a package from using a package for two main reasons:
  • Multiple Targets
    • A single CMakeLists.txt file might build five different executables or libraries. If find_package automatically applied Eigen to everything in the file, it would cause massive bloat and unnecessary dependencies for targets that don't need it.

  • Scope Control (Public vs. Private)
    • CMake needs to know how you are using Eigen. In your code, you used PUBLIC. This tells CMake: "I need Eigen to build this library, and anyone who links against my library also needs Eigen." If you used PRIVATE, it would mean: "I use Eigen internally, but hide it from anyone using my library." find_package cannot guess your architectural intent.
