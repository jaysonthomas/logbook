Cmake Commands: Debug & Assignment

Debug

CMake provides several log levels to categorize a message:
  • STATUS: Normal informational messages.
  • WARNING: Warnings that do not stop the build process.
  • ERROR: Errors that will terminate the configuration process.
  • FATAL_ERROR: Errors that immediately terminate the CMake process.
  • DEBUG: Messages shown only when CMake is run in debug mode.

Example


message(STATUS "The value of MY_VAR is: ${MY_VAR}")

Setting variables


set(PACKAGE_NAME matplotlib_cpp)
set(INSTALL_CONFIGDIR ${CMAKE_INSTALL_LIBDIR}/${PACKAGE_NAME}/cmake)
  • PACKAGE_NAME
    • It is is the name of the package being built and installed.

  • INSTALL_CONFIGDIR
    • It is the location where CMake configuration files for the package will be installed, for other libraries to be able to find it. It's defined as a subdirectory under the cmake-library installation directory.

    CMAKE_INSTALL_LIBDIR is /lib.
