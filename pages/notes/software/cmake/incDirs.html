Cmake commands: Target include directories

Overview

target_include_directories specifies the include directories to use when compiling a given target.

To include header files present in the source directory of a target library b, where the CMakeLists.txt file is also stored: 


target_include_directories(
  b PUBLIC
  ${CMAKE_CURRENT_SOURCE_DIR}
)
This allows any other target that links against b to find its headers.

To include files from a directory util in the root project directory of a library: 

target_include_directories(
  b PUBLIC
  ${CMAKE_SOURCE_DIR}/util/
)

Private

If PRIVATE was used to include a library's headers within itself, these headers wouldn't be visible to a project even if the library was linked to within the project.

PUBLIC or INTERFACE would need to be used explicitly. In this case, linking will automatically propagate the library's include directories.
