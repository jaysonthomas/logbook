Using python with drake

Issue: python confusion

Issue

Even though yaml was installed system wide using sudo apt install python3-yaml, bazel was still giving out an error: ModuleNotFoundError: No module named 'yaml'.

Solution

Bazel uses its own sandboxed environment, which means any system Python won't be visible to it by default unless explicitly made part of the build.

The best practice is to declare dependencies with rules_python.

Under drake_bazel_external, look at setup->install_prereqs to understand what python version is used by drake by default.
