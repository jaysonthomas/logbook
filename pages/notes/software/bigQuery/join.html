Inner join

INNER JOIN or JOIN, calculates the Cartesian product of the two from_items and discards all rows that don't meet the join condition. [Ref].

Imagine 2 tables with fields:

  • Table 1: startTime, stopTime, id
  • Table 2: time, error
When joining based on the condition that table2.time needs to be between table1.startTime and table1.stopTime, a new virtual table is created where all the rows of the 2nd table that meet this condition are joined with the fields of table 1. Considering just the first row of table 1, this means that the fields of this row are duplicated and attached to every single row of table 2 that satisfies the condition. 


SELECT
  t2.id, t2.time, t2.error,
  # ROW_NUMBER usage
FROM
  table1 AS t1
JOIN
  table2 AS t2
ON
  t1.id = t2.id
WHERE
  t2.time BETWEEN t1.startTime AND t1.stopTime
Here, the join happens first (based on ON condition), then the filtering. Since, this is an inner join, the SQL Query Optimiser, combines the 2 operation. It would be the same as having all the conditions under ON. This wouldn't be the case with a left join.
