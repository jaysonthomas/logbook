Polymorphism

Overview

It means "assuming many forms". It describes a paradigm in which a function may behave differently depending on how it is called, i.e. based on its inputs. It allows an interface to work with several different types. We might have a cut function that can work with the plant, the fruit, or the apple class. On the other hand, we might have a brews function that works with fruits and with apples, but not with plants.

Polymorphism can be achieved in different ways, the 2 main ones being overloading and overriding.

Overloading

Function overloading

[Ref]
Same function name, but different function signature, i.e. datatypes of input arguments.

Operator overloading

[Brackets, Addition, Dereferencing]
In order to overload an operator, the operator keyword is used in the function signature. Eigen is a matrix library in cpp.

Overriding

[Ref]
Overriding a function occurs when:
  • A base class declares a virtual function
  • A derived class can override the virtual function by defining its own implementation with an identical function signature
Specifying a function as override in the derived class is good practice, as it empowers the compiler to verify that the function signature in the derived class matches that in the base class.

Generic programming

Generic Programming is an example of polymorphism that generalises the parameters of classes and functions using templates. The aim is to create code (classes and functions) that can be reused with many different types of data. For example using the vector class from the Standard Library, a vector of ints, strings or some user defined class can be created.

Templates

[Function template, Class template]
Templates use datatypes as parameters so that the same implementation can operate on different data types. At compile time, the compiler expands the code using the types that are passed as parameters. 


// If a class
template <typename dataType1, typename datatype2>
class X {
  X(datatype1 key, datatype2 value)
  {
    // Constructor
  }
};

// Calling the class
X<sting, int>("age", 3);

// If a function
template <typename datatype>
datatype X(datatype input1, datatype input2)
{

}

// Calling the function
X<int>(5, 6);
Generic code is the term for code that is independent of types. A template<> tag is placed before the function/class signature to specify and mark the declaration as generic. The keyword typename (or alternatively, class) specifies the generic (data)type which the parameter that follows it, takes on.

Virtual functions

[Ref]
Virtual functions are a polymorphic feature. These functions are declared (and possibly defined) in a base class, and can be overridden by derived classes. This approach declares an interface at the base level, but delegates the implementation of the interface to the derived classes.

Virtual functions can be defined by derived classes, but this is not required. However, if we mark the virtual function with =0 in the base class, then the function becomes a pure virtual function. This means that the base class declares but does not define the function. It has the side effect of making its class abstract meaning the class cannot be instantiated. A derived class must define the function, or else the derived class will be abstract.

We don't know how a generic animal talks. So we declare a function talk as a virtual function. =0 means we're never going to create an object of type Animal. We're simply creating an interface such that other classes that inherit from this base class will have to implement the talk function.
