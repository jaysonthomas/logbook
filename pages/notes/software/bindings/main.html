Bindings

Overview

A binding is a way to connect code written in one programming language with that of another, i.e. to call functions from one language to another.

Java to C

Java runs on the Java Virtual Machine (JVM) and does not execute native code directly. To interact with C code, Java provides mechanisms like:
  • Java Native Interface (JNI) - The official way to call C (or C++) code from Java.
  • Java Native Access (JNA) - A higher-level, easier-to-use alternative to JNI.

Steps using JNI

  • In a java file, load the library with its name. A load function is provided by the binding mechanism
  • Compile the Java file
  • Generate a c header file using the binding mechanism
  • Implement the C function in a C file and include the generated header file. Also include header files from the binding mechanism. The return value and the arguments to the C function may have the java-binding types.
  • Compile the C code into a shared library
  • Run the java code.

Steps using JNA

  • Create the C file and compile into a shared library
  • Write the java code and load the c library using jna binding mechanism
  • Run the java program; the shared library needs to be in the same directory as the java program or in a system library path.
JNAerator is a tool that automatically converts C/C++ header files (.h) into Java interfaces that work with JNA.
