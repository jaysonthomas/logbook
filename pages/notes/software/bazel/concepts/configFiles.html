Bazel concept: Configuration files

The main one

.bazelrc is a configuration file that specifies default build options for Bazel. It helps control things like:
  • Compiler options
  • Remote caching
  • Repository overrides
  • Execution strategies

User specific

The meaning of try-import %workspace%/user.bazelrc:
  • try-import tries to load user.bazelrc, but doesn't fail if the file is missing.
  • %workspace% refers to the root of the Bazel workspace - user.bazelrc is expected to be in the same directory as MODULE.bazel.
  • This allows users to have personal Bazel settings without modifying the main .bazelrc

Useful commands

  • Check if user.bazelrc is loaded
    • 
bazel help | grep 'override_module'
    If it works, the output should be what was specified in the user file.
