SQL

Aggregate

Aggregate functions like COUNT(), SUM(), AVG(), MAX() etc. instructs SQL to reduce multiple rows into a single summary value (or one per group). But SQL needs clear instructions - if rows are being summarised based on 1 column, what should it do with the other columns?

Rule

Every column in the SELECT clause must either be:
  • An aggregate function or
  • Listed in the GROUP BY clause

Join

[Ref]

(INNER) JOIN enforces a strict match between the rows of the tables mentioned in the FROM clause. Else, the rows are discarded.

LEFT JOIN, retains the left side no matter what. If there is no RHS, then the RHS part of the row is left blank.

Lead


LEAD(timestamp, 1) OVER (PARTITION BY id ORDER BY timestamp ASC) AS nextTs
The above query partitions the huge table by id. Each partition is then sorted by the timestamp column in an ascending order. A new column is then added to the table called nextTs where each entry in it is the next entry of the timestamp column.

This is mainly used in the SELECT clause of a query.
