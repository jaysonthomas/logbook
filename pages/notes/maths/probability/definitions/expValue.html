Expected value

Intuition

It is not the most likely, but the average value a random variable converges to if a value was continuously drawn from a distribution. It gives the long-run average value of a repeated probabilistic experiment.

Mathematical expression

$$ E(X) = \sum_{i=1}^{N}v_iP(X=v_i) $$ It's the value of the random variable, times the likelihood of each of those outcomes all summed together.

Random variable

Where the underlying events are non-numeric quantities such as working or not working, the expected value of a random variable may not be useful because the numeric values assigned to the random variables wouldn't make sense.

Where the underlying events are numeric like the position of a bot, the expected value gives an idea of the value of the random variable at any given time.

Special cases

Variance

$$\begin{align*} \sigma^2 &= E[X = (v_i - \mu)^2] \\ &= \sum_{i=1}^{N}(v_i-\mu)^2P(X=v_i) \end{align*}$$ $P(X=v_i)$ is the same as $P(X = (v_i - \mu)^2)$.

$\sigma^2$ is the long-term average ($E$) of the square of the deviation from the mean.

Equal probability

When the probability of each event is $\frac{1}{N}$, the expected value is equal to the mean of the distribution.

Trace trick

The dot product within the expectation $E\left[(x - \hat x)^\top (x - \hat x)\right]$ results in a scalar. Wrapping the scalar in a trace operator means nothing - trace of a scalar is the same scalar. $$\begin{align} E \left[tr((x - \hat x)^\top (x - \hat x))\right] &= E \left[tr((x - \hat x) (x - \hat x)^\top)\right] \label{eq:1} \\ &= tr \left(E \left[(x - \hat x) (x - \hat x)^\top\right]\right) \label{eq:2} \\ &= tr(\Sigma) \nonumber \end{align}$$ Permuting the entries inside a trace makes no difference as in \ref{eq:1}. Instead of a dot product, it is now an outer product. The expectation of the outer product is the covariance.

Expectation and trace are linear operators. Since both are linear, the order of the operations does not matter. Moving the trace outside as in \ref{eq:2}.
