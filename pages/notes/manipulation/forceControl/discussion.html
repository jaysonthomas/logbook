Use case

  • Metal grinding
    • In metal grinding for a good surface finish, a force-torque sensor at the end of an arm would be good. But, a good model can also be built using just motor current.

  • Erasing/writing on a board
    • Exact positions are not know to set target points, but the force to be applied on the board is known.

  • Screwing a bulb
    • The angular rotation is not known to tighten the bulb, but the screwing should stop when the required amount of torque limit is reached.

  • Non rigid body handling
  • Peg in the hole
    • The usual approach would be to use a vision system to detect the location of the hole. But, there is a high chance of collision and damaging a surface's finish.

All these tasks could be done with conventional position or velocity based controllers, but they would need to be augmented with additional sensors like vision systems etc.

There are 2 main kinds of control methodologies: velocity and force control.

Velocity control

$$ \dot\theta(t) = \dot\theta_d(t) + K_p \theta_e(t) + K_i \int^t_0 \theta_e(t) dt $$ For each joint, an error in the position can be computed from the desired and the current positions. Proportional and integral gains can be used on the error to command a velocity input to each joint which would get any particular joint to go from the current to the target position.

This works for stepper motors where changing the frequency of the input pulse changes the rotor velocity. Position encoders can provide feedback to close the control loop.

The above equation can also be written in terms of body twist to translate it into the task space. $$ V_b(t) = [Ad_{X^{-1} X_d}] \, V_d(t) + K_p X_e(t) + K_i \int^t_0 X_e(t) dt $$ The desired twist can be computed from the end effector's current and target positions.

Usually, all this is handled by a low level controller. This controller can be written at a higher level, but it would require for the lower level controller to accept joint velocity rather than joint position set points.

Impedance control

In motion-force control, the aim is to avoid arbitrary motions, but have arbitrary forces at the end effector. Impedance control avoid arbitrary motions due to arbitrary disturbing forces/perturbations.
