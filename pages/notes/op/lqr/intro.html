Introduction

Overview

LQR problem is a trajectory optimisation problem AKA deterministic optimal control problem.

Linear dynamics is assumed. Cost functions/weights on both the states and the controls are quadratic and can vary with time. There's a terminal cost on the final state. $$\begin{align*} \min_{\substack{x_{1:N} \\ u_{1:N-1}}} \quad & J = \sum_{k=1}^{N-1} \frac{1}{2} x_k^\top Q_k x_k + \frac{1}{2} u_k^\top R_k u_k + \frac{1}{2} x_N^\top Q_N x_N \\ s.t. \quad & x_{k+1} = A_k x_k + B_k u_k \\ & Q \geq 0, \quad R > 0 \end{align*}$$ The dynamics also vary with time, hence $A,B$ matrices have $k$ subscripts. The problem is time invariant if $A_k=A, B_k=B, Q_k=Q, R_k=R \; \forall k$.

For a QP to be convex, the Hessian has to be positive definite. Similarly, the state cost matrix $Q$ needs to be positive semi-definite and the control cost matrix $R$ needs to be strictly positive definite for the problem to have a well posed solution.

If $Q=0$, it would mean certain states are ignored/not important.

If $R=0$, there's no cost on having control which can lead to an unbounded control solution, i.e. the controls could blow up to infinity. If there is a cost to getting to a goal and there is no cost on the controls, the solution could simply slam the controls to infinity and get there instantly which would be ill-posed, infinite solution.

The initial state is always given.

The only constraints allowed for an LQR problem are those for the dynamics.

Advantages

  • The reason LQR is so useful is because most nonlinear problems can be locally approximated as an LQR one. This is useful in creating local tracking controllers around trajectories and stabilizing controllers around equilibrium points.

  • Computationally tractable even with problems with states of many dimensions. Able to solve them quickly.

  • Can be applied to a variety of problems like stochastic, partially observable, infinite horizon problems etc.

  • LQR is a rare gem where there's a closed form solution. It is possible to get a lot of insight and leverage that closed form solution in different areas. When other constraints are added in, it ends up not having closed form solution. It would then need to be solved numerically.
