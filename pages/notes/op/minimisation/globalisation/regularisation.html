Regularisation

Overview

Hard linear problems or nonlinear problems are usually not strongly convex. Regularisation is a trick used to minimise a function that is not strongly convex.

The Hessian needs to be positive definite to guarantee a descent step. If it is not, it is padded with 0s until it is.

Also called Damped Newton method. If $\nabla^2 f$ is made bigger, then the inverse becomes smaller. Hence, the step size or learning rate shrinks which leads to damped/shorter steps.

Steps

Initialise some variable $H$ with the Hessian matrix: $$ H \leftarrow \nabla^2 f $$ While $H$ is not positive definite, it is padded with some multiple of the identity. $$\begin{align*} &\text{while } H \ngtr 0 \\ &\qquad H \leftarrow H + \beta{I} \\ &\text{end} \end{align*}$$
$$\begin{array}{ccl} \beta &:& \text{Scalar hyperparameter that can be tuned. } \beta > 0\\ \end{array}$$
Once $H$ is positive definite, run the Newton step: $$\begin{align*} \Delta{x} &= - H^{-1} \nabla{f(x)} \\ x &\leftarrow x + \Delta{x} \end{align*}$$ In practice, a custom cholesky factorisation is attempted that introduces padding on the fly as it runs into negative values. Specifically, cholesky computes the square roots of the diagonal and if it comes across a negative eigenvalue somewhere, it will end up with an imaginary number and that's how it knows it failed. It then just pads that spot and keeps going.

Intuition

The Hessian of the objective function ($H$) must be positive definite for the problem to have a unique, stable minimum. If the Hessian is "flat" or has negative curvature (meaning it looks like an upside-down bowl), the solver will get lost.

Adding $\beta I$ pulls the edges of the upside-down bowl upwards, turning it into a right-side-up bowl. Mathematically, this shifts all eigenvalues of the Hessian to the right.

If an eigenvalue was at $-0.5$ and you add $\beta = 1.0$, the new eigenvalue becomes $+0.5$.

Performance

Regularisation makes sure each solution heads downhill and never uphill. It still exhibits the overshoot problem though.

If the initial guess is exactly at a maximum, the solution will just stay there.
