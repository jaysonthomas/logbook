Regularisation

Overview

Hard linear problems or nonlinear problems are usually not strongly convex. Regularisation is a trick used to minimise a function that is not strongly convex.

If the Hessian is not positive definite, it is padded with 0s until it is and then that's going to guarantee a descent step.

Also called Damped Newton method. If $\nabla^2 f$ is made bigger, then the inverse becomes smaller. Hence, the step size or learning rate shrinks which leads to damping the steps.

Steps

Initialise some variable $H$ with the Hessian: $$ H \leftarrow \nabla^2 f $$ While $H$ is not positive definite, it is padded: $$\begin{align*} &\text{while } H \ngtr 0 \\ &\qquad H \leftarrow H + \beta{I} \\ &\text{end} \end{align*}$$
$$\begin{array}{ccl} \beta &:& \text{Provides padding of the Hessian matrix with some multiple of the identity.}\\ && \text{Scalar hyperparameter that can be tuned.} \\ && \beta > 0 \end{array}$$
Once $H$ is positive definite, run the Newton step: $$\begin{align*} \Delta{x} &= - H^{-1} \nabla{f(x)} \\ x &\leftarrow x + \Delta{x} \end{align*}$$ In practice, a custom cholesky factorisation is attempted that introduces padding on the fly as it runs into negative values. Specifically, cholesky computes the square roots of the diagonal and if it comes across a negative eigenvalue somewhere, it will end up with an imaginary number and that's how it knows it failed. It then just pads that spot and keeps going.

Shortcomings

Regularisation makes sure each solution heads downhill and never uphill. It still exhibits the overshoot problem though.

If the initial guess is exactly at a maximum, the solution will just stay there.
