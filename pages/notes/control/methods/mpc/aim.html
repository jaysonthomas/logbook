Notation

Prediction horizon

It is the future horizon over which the state trajectory is predicted.
$$\begin{array}{ccl} p &:& \text{Length of horizon} \\ x_{k+i|k} &:& \text{Prediction of the state vector for the time instant }k+i \text{ made at }k \text{. } \\ z_{k+i|k} &:& \text{Prediction of the output vector for the time instant }k+i \text{ made at }k \text{. } \\ i=0,1,2,\cdots,p && \end{array}$$

Control horizon

It is the future horizon over which the control inputs are allowed to change.
$$\begin{array}{ccl} c &:& \text{Length of horizon} \\ u_{k+i|k} &:& \text{Control input at }k+i \text{ computed at }k \text{. } \\ i=0,1,2,\cdots,c && \end{array}$$
$$ \text{Limitation: } c \leq p $$ The control input is determined as the solution to the MPC problem. It only varies until $k+c$, after which it is constant and equal to the last value in the control horizon $u_{k+c|k}$.

Statement

The goal of MPC is to determine a set of control inputs $u_{k+i|k}$ at time instant $k$, that will make the output of the system $z_{k+i|k}$ follow a desired control trajectory over the prediction horizon.

This is done by:

  • Using the knowledge of the current state $x_k$, and the system model matrices $A,B,C$ to predict the output trajectory $z_{k+i|k}$
  • Minimising the difference between the desired and predicted trajectories
