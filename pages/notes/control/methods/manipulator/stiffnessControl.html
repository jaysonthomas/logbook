Stiffness control

Overview

The philosophy is for a robot to have a programed response to torque perturbations (if it was pushed or if it ran into something), instead of a rigid position control. A robot can be made to respond gracefully as if it were a different mechanical system, say a mass-spring-damper system: $$\begin{equation} m\ddot{x} + b\dot{x} + kx = f \end{equation}$$ This control technique is also referred to as simple impedance control. Impedance control is the more general term for the whole set of concepts.

Overview

A big complicated robot can be made to act like a point robot at the end using a stiffness controller, wherein the end-effector's interactions to external forces are programed to be that of a mass-spring-damper.

A stiffness controller can also be created to replace the effect of gravity (i.e. gravity compensation).

Benefit of having gravity compensation

In practice, the difference in having a controller with just PD control and no gravity compensation is huge.

If reasonable performance is desired without gravity compensation all over a system's state space with just a PD controller, $K_p$ would have to be very large. A large $K_d$ would also be needed to damp the effect of having a large $K_p$. If gravity is subtracted i.e. GC is used in the controller, then $K_p, K_d$ can be chosen to be much lower and the system can be much more responsive.

In practice, when a robot not running GC gets bumped into, it may trigger a fault and power down. Using GC, would make the robot react soft enough to get out of the way.
