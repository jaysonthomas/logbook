Linear systems

Overview

$$ \dot{x} = A(t) \, x + B(t) \, u $$ The system is called linear time invariant if $A(t) = A$ and $B(t) = B$, i.e. constants. It is called linear time varying otherwise. Compared to control affine, this is linear in both $u$ and $x$.

If a system's dynamics are linear, the control problem can be written as a convex optimisation problem.

Linearising a non-linear system

If a system is non-linear, as long as it's smooth, it's derivatives can be taken and linearised locally to then approximate the system as a linear one. This works 90% of the time. The linear system can now be useful for analysis and to design local controllers.

The system $\dot{x} = f(x, u)$ can be approximated locally using a linear system about some nominal $x, u$ by taking the jacobians, i.e. by assigning $A = \frac{\partial{f}}{\partial{x}}$, $B = \frac{\partial{f}}{\partial{u}}$.

State transition function

[Ref]

$$\begin{align*} z_k &= Cx_k \\ \dot{x}_{k+1} &= Ax_k + Bu_k \end{align*}$$
$$\begin{array}{ccl} A &:& \text{System matrix - maps how a state relates to the next state} \\ B &:& \text{Input matrix - maps how a control input relates to the next state} \\ C &:& \text{Output matrix - maps a state to the measurement space} \end{array}$$
Consider a non-simulation environment, at time instant $k$, a prediction $z_k$ is made of the actual measurement $y_k$ that is obtained of the state at $k$. In a simulation environment, this would serve as the measurement.

Once the control input to be applied $u_k$ is decided based on the current state measurement, a prediction can be made of what the state would look like in the next time instant, $x_{k+1}$.

$y_k$, $z_k$ point to the state variables that the controller aims to control to follow a desired trajectory. It is not necessarily the observed output, that is, it is not necessarily directly measurable.
