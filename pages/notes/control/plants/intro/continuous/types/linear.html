Linear systems

Continuous time

$$ \dot{x} = A(t) \, x + B(t) \, u $$ The system is called linear time invariant if $A(t) = A$ and $B(t) = B$, i.e. constants. It is called linear time varying otherwise. Compared to control affine, this is linear in both $u$ and $x$.

If a system's dynamics are linear, the control problem can be written as a convex optimisation problem.

Discrete time - state transition function

[Ref] $$\begin{align*} z_k &= Cx_k \\ x_{k+1} &= Ax_k + Bu_k \end{align*}$$
$$\begin{array}{ccl} A &:& \text{System matrix - maps how a state relates to the next state} \\ B &:& \text{Input matrix - maps how a control input relates to the next state} \\ C &:& \text{Output matrix - maps a state to the measurement space} \end{array}$$
Consider a non-simulation environment. At time instant $k$, a prediction $z_k$ is made of the actual measurement $y_k$ that is obtained of the state at $k$. In a simulation environment, this would serve as the measurement.

Once the control input to be applied $u_k$ is decided based on the current state measurement, a prediction can be made of the state in the next time instant, $x_{k+1}$.

The controller aims to control the state $x_k$ to follow a desired trajectory. It is not necessarily observable, i.e. directly measurable.

Linearising a non-linear system

The system $\dot{x} = f(x, u)$ can be approximated locally using a linear system about some nominal $x, u$ by taking the jacobians, i.e. by assigning $A = \frac{\partial{f}}{\partial{x}}$, $B = \frac{\partial{f}}{\partial{u}}$. Here higher order terms are ignored.

If a system is non-linear, as long as it's smooth, its derivatives can be linearised locally to approximate the system as a linear one. This works 90% of the time. The linearised system can then be useful for analysis and to design local controllers.
