Anaotmy of a robot task - forward model

$$ v = J(q) \dot{q} $$
$$\begin{array}{ccl} v \in R^m &:& \text{Cartesian velocities (twists)} \\ J(q) &:& \text{Robot Jacobian} \\ q \in R^n &:& \text{Joint velocities} \\ n \geq m &:& \text{Dimension of the joint velocities is greater than that of the task} \\ \end{array}$$
A task is represented by cartesian velocities (here, a twist with 6 DOF). Each twist is related to a joint velocity through the robot Jacobian. Ideally, the dimension of a task $m$ is much greater than $n$, so that there is some redundancy and therefore lots of interesting ways to achieve it.

Aim

Given the task $v$ in cartesian space, find the vector of joint velocities $\dot{q}$.

This can be formalised as a least squares problem - the aim is to minimise the difference between the task vector $v$ and the model that maps to the cartesian space: $$\begin{equation} \min_{\dot{q}} \quad \|v - J\dot{q}\|^2\\ \label{eq:formalAim} \end{equation}$$ $\min_{\dot{q}}$ means minimise $\dot{q}$.

Particular solution

The particular solution of diff IK inovles doing the algebraic inversion of a linear mapping. $$\begin{align} \dot{q}^* &= J^T \, (JJ^T)^{-1} v \label{eq:jacobInverse} \\ &= J^\dagger \, v \label{eq:pseudoInverse} \end{align}$$ This is the right pseudo-inverse of the robot Jacobian solution. It can be inspected that substituting $\ref{eq:jacobInverse}$ into the problem's objective function $\ref{eq:formalAim}$ will minimise the difference. This is how the inversion $\ref{eq:pseudoInverse}$ is arrived at.

This method is equivalent to finding 1 particular point within what could be, in general, a large subset of potential solutions.

Homogeneous solution

Adding a second component to the particular solution leads to the homogeneous solution to the problem. It is essentially a vector of joint velocities $q_0$ mapped into the null space of the Jacobian using $I - JJ^\dagger$ $$\begin{equation} \dot{q}^* = \underbrace{J^\dagger \, v}_{\text{Particular} \atop \text{component}} + \underbrace{(I - JJ^\dagger) \, q_0}_{\text{Homogeneous} \atop \text{component}} \label{eq:homSol} \end{equation}$$ Whatever is being mapped (here, $q_0$) needs to be projected into a space where it doesn't affect the main task $v$. The homogeneous component does not affect the task, but allows projecting secondary tasks into the null space.

With the homogeneous formulation, any point can be constructed within the green valid solution set for the diff IK problem.

Example

This video is from a paper that's unrelated to diff IK, but nicely shows what a null space solution means.

The task is to move the tip of an end effector along a hexagon. This can be solved in many ways if the robot has more DOFs than the DOFs needed to specify the task. Here, the task has 3 DOFs and the robot has 7, so the null space $I - JJ^\dagger$ is 4 dimensional. The robot joints seem to bounce/dance in the null space.

These kind of motions can be chosen arbitrarily as long as they don't conflict with the primary task which is achieved with the null space projection.

Limited DOFs

Minimising the solution space $I - JJ^\dagger$ is equiavalent to minimising the difference to 0. Is there a case where it cannot be minimised to 0?

If the joint space does not have as many DOFs as needed to accomplish a task. E.g., consider a robot with only 4 joints trying to accomplish a 6 DOF cartesian task/twist.

In this case, it wouldn't be possible to minimise the cost function. The end solution would be the best that could be found in a least squares sense to the problem. It will accomplish as much as it can without actually solving the problem of executing the task perfectly.

In the case of the humanoid, it might not be possible to achieve all of the tasks when there are multiple conflicting objectives. A hint at that is also the null space element. The null space will be done as well as possible as long as it doesn't conflict with anything that lives in the particular solution side of the equation.

Getting stuck in local minima

Some basic IK solvers like closed loop IK, will iterate on equations of type \ref{eq:homSol} until they converge to a joint solution. They can get stuck.

More advanced IK solvers like track IK, will do many of these in parallel threads and then even a QP solution on top. Here, lots of local searches and random restarts are done to avoid the problem of local minimums.

All the diff IK discussed here is local.
