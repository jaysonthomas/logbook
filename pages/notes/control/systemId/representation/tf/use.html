TF: Use

Overview

The response of a system to different input forces can be found by representing the input in the s domain and multiplying it with the system's transfer function.

The real power with transfer functions goes beyond using them to apply different forcing functions to a system. It is that we can use them to represent very complex systems by building them out of smaller, less complex systems.

Output divided by input

Example

Imagine, the transfer function of a cart system is to be determined. For which a test is performed by applying a step function to it and measuring the resulting motion. The s domain equation that produces a similar motion is found to be: $$ \frac{1}{s(ms^2 + bs + k)} $$ But, this is not the transfer function of the cart itself because it's not the impulse response; it's the step response. Dividing the output by the input, the cart's transfer function can be found: $$ \frac{\frac{1}{s(ms^2 + bs + k)}} {\frac{1}{s}} = \frac{1}{ms^2 + bs + k} $$

Misconception

Defining a transfer function as the output of a system divided by its input in the s domain, is wrong. It is true that this produces the same transfer function as the Laplace transform of the impulse response, however, it is misleading to think that it is the definition of a transfer function.

This is because it hides what really makes a transfer function work; namely, that it's performing the convolution of the input and the impulse response.
