Big O Notation

Overview

Big O time complexity is a way of analysing the amount of time it takes for an algorithm to execute as the input size grows. The worst case is always considered.

Any constants are ignored apart from $1$.

O($1$)

An algorithm that always executes in the same time regardless of the size of the data set. Efficient with any data set.

Example use:

  • Extracting data from any element from an array.
  • Hashing algorithm.

O($log \, N$)

Logarithmic. An algorithm that halves the data set in each pass. Opposite to exponential. Efficient with large data sets.

Example use:

  • Binary search.
  • Push/pop on a heap.

Binary search

Consider an array of size $N$. How many times can the array be divided by 2 to get just 1 element. In other words, how many times can 1 be multiplied by 2 to get $N$. In other words, 2 to the power of what gives $N$, which is the definition of log. $$\begin{align*} 2^x &= N \\ x &= log_2 N \end{align*}$$ An array of size N can be divided $log_2 N$ times. The base here is 2.

O($N$)

In this case, as the input size grows, the execution time grows proportionately.

Example use:

  • A loop iterating through a single dimension array (e.g., sum of array values).
  • Linear search.

O(N $log \, N$)

Algorithms that divide a data set but can be solved using concurrency on independent divided lists.

Example use:

  • Quick/Merge sort.

O($N \cdot M$)

Very similar performance to O($N^2$)

Example use:

  • 2D rectangular matrix.

O($N^2$)

In this case, as the input size grows, the performance is proportional to the square of the size of the data set.

Example use:

  • A nested loop iterating through a two dimension array.
  • Bubble sort.
